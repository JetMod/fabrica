# Инструкция к сайту-каталогу «ФАБРИКА интерьера»

## Назначение сайта

Сайт создаётся как каталог дизайнерской мебели и интерьерных решений с функцией презентации бренда и сбора заявок.

**Цель сайта:**
- Продемонстрировать дизайнерскую мебель и комплексные интерьерные решения
- Показать экспертность компании и производство полного цикла
- Привлечь дизайнеров, девелоперов и частных клиентов
- Упростить путь клиента от просмотра каталога до заявки

**Важно:** Сайт не является интернет-магазином. Основная функция — витрина + лидогенерация.

### 1. Дизайнеры интерьера

### Именование классов
Рекомендуется использовать методологию **BEM** (Block Element Modifier) для удобной интеграции в WordPress:

```html
<!-- Блок -->
<div class="card">
  <!-- Элемент -->
  <img class="card__image" src="...">
  <h3 class="card__title">...</h3>
  <!-- Элемент с модификатором -->
  <button class="card__button card__button--primary">...</button>
</div>
```

**Правила:**
- Блоки: `.block`
- Элементы: `.block__element`
- Модификаторы: `.block--modifier` или `.block__element--modifier`
- Избегать вложенности более 2-3 уровней

## Требования к изображениям

### Форматы
- **Фото** — WebP (с fallback на JPG)
- **Иконки, логотипы** — SVG
- **Графика** — PNG (если нужна прозрачность)

### Размеры и оптимизация
- **Hero-изображения** — до 1920px по ширине, оптимизированы
- **Карточки товаров** — 800-1200px по ширине
- **Иконки** — SVG, оптимизированные
- **Lazy loading** — для всех изображений ниже fold
- **Responsive images** — srcset для разных разрешений

### Правила
- Все изображения должны иметь alt-атрибуты
- Оптимизация через инструменты (TinyPNG, ImageOptim)
- Размер файла: фото не более 200-300KB

## Анимации и переходы

### Принципы
- Плавные, ненавязчивые анимации
- Длительность: 200-300ms для простых, до 500ms для сложных
- Easing: ease-in-out или cubic-bezier для естественности

### Где использовать
- Hover-эффекты на кнопках и карточках
- Появление элементов при скролле
- Переходы между состояниями форм
- Открытие/закрытие модальных окон
- Мобильное меню

### Что избегать
- Излишние анимации
- Медленные переходы
- Анимации, мешающие восприятию контента

## Сетка и отступы

### Система отступов
- Базовый шаг: **8px** или **4px**
- Отступы между секциями: 80-120px (десктоп), 40-60px (мобильный)
- Отступы между элементами: 16-32px
- Внутренние отступы карточек: 24-32px

### Контейнеры
- Максимальная ширина контента: 1200-1400px
- Отступы по бокам: 20-40px (мобильный), 40-80px (десктоп)
- Центрирование контента

## Интеграция с WordPress

### Подготовка к интеграции
- Компоненты должны быть выделены в отдельные блоки
- Повторяющиеся части (header, footer) — в отдельные файлы
- Формы — готовы к подключению WordPress-функций
- Изображения — пути относительные, легко заменяемые на `wp_get_attachment_image()`

### Структура для WordPress-темы
- Шаблоны страниц (page templates)
- Кастомные типы постов (Custom Post Types) для каталога
- Таксономии для категорий каталога
- Поля ACF (Advanced Custom Fields) для дополнительных данных

### JavaScript модули
- Каждый модуль — отдельный файл
- Использование `DOMContentLoaded` для инициализации
- Избегать глобальных переменных
- Готовность к подключению через `wp_enqueue_script()`

## Примечания

- Все изображения должны быть оптимизированы
- Использовать современные веб-стандарты
- Код должен быть читаемым и поддерживаемым
- Документировать сложные решения
- При разработке учитывать будущую интеграцию в WordPress
- Использовать чистый JavaScript без зависимостей
- Структура кода должна позволять легко выделить повторяющиеся компоненты для WordPress-шаблонов
- Тестировать на реальных устройствах, не только в DevTools
- Проверять производительность (Lighthouse)
